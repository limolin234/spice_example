Meanings:

- `cir/`: hand-written circuit topology and reusable subcircuits for the case.
- `cir/main.par`: circuit parameters, usually generated by Python.
- `lib/`: external models and vendor libraries, not local hand-written circuit hierarchy.
- `sim*/tb.cir`: complete testbench: includes parameters, declares excitation sources, instantiates the circuit, selects analysis, and writes output.
- `sim*/tb.par`: testbench parameters, including analysis parameters and source parameters, generated by Python.
- `sim*/output/`: local output for that test.
- `run.sh`: minimal launch glue using the `sci` micromamba environment.

Run paths are fixed: run from the case root. `.include` paths are written relative to the case root.

Different test types should use subdirectories such as `sim/ac/`, `sim/dc/`, or `sim/tran/` rather than splitting sources out of `tb.cir`.
